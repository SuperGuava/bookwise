# BookWise Product Requirements Document (PRD)

## 목표 및 배경 컨텍스트

### 목표

- 외부 관리자 페이지에서 등록된 전자책을 신속히 BookWise 메인 스토어에 노출하여 글로벌 구매자를 확보한다.
- 카드 기반 상품 목록과 상세 페이지를 통해 구매자가 상품 비교부터 결제 페이지 이동까지 최소한의 단계로 완료하도록 한다.
- 결제 흐름과 트래픽 데이터를 단순한 파이프라인으로 유지해 운영자가 글로벌 판매 성과를 확인할 수 있도록 한다.

### 배경 컨텍스트

BookWise는 전자책 중심의 디지털 상품 판매 수요가 급증하는 가운데, 글로벌 고객을 겨냥해 운영자가 외부 관리자 페이지에서 등록한 콘텐츠를 실시간으로 노출하고 구매로 연결하는 SaaS 기반 솔루션이다. 기존 마켓플레이스는 입점 절차 및 수수료 정책이 과도하고, 자체 웹사이트를 구축하는 운영자는 결제‧보안‧배포 인프라 운영 부담이 크다. BookWise는 콘텐츠 노출부터 구매 연동까지 한 번에 제공하여 이러한 지점을 해결한다.

## 요구사항

### 기능 요구사항 (Functional Requirements)

#### 콘텐츠 등록 및 관리

- **FR001**: BookWise 메인 스토어는 Lemon Squeezy Store에 등록된 상품 목록을 API로 가져와 카드 형태로 노출하며, 각 카드에서 제목·설명·가격을 확인할 수 있어야 한다.
- **FR002**: 메인 페이지는 제목 또는 설명 기준 검색 기능을 제공해 구매자가 원하는 전자책을 빠르게 선별할 수 있어야 한다.
- **FR003**: 외부 관리자 페이지의 상태 변경(노출 여부, 프로모션, 번들 옵션 등)은 BookWise에 자동 반영되며, 각 카드에 최신 상태 태그를 표시해야 한다.

#### 판매 경험 및 수익화

- **FR004**: BookWise 메인 스토어는 카드 목록을 통해 콘텐츠를 소개하고, 카드 및 상세 페이지의 결제 버튼을 외부 결제 페이지로 이동하는 단순한 구매 경험을 제공해야 한다.
- **FR005**: 상세 페이지는 Lemon Squeezy 상품 설명·가격·정적 FAQ를 제공하고, 추천 상품 영역을 통해 관련 콘텐츠를 제안해야 한다.
- **FR006**: 외부 결제 리턴 데이터를 수집해 운영자가 노출 전략과 콘텐츠 우선순위를 조율할 수 있어야 한다.

### 비기능 요구사항 (Non-Functional Requirements)

- **NFR001**: 결제‧파일 다운로드‧상품/상세페이지 요청에 대해 평균 응답 시간을 300ms 이하로 유지하며 사용자 지연을 최소화한다.
- **NFR002**: 콘텐츠 파일은 DRM 또는 서명 기반 검증을 거쳐 전달하며, 모든 트래픽은 TLS로 암호화한다.
- **NFR003**: 아시아, 북미, 유럽 등 주요 지역 사용자 증가에 대비해 멀티 리전 CDN과 백업 정책을 구성하고, 다국가 결제·통화·세금 흐름을 원활히 처리하며 월간 트래픽 100K 세션을 안정적으로 소화할 수 있도록 한다.

## 사용자 여정 (User Journeys)

### 여정 1: 구매자가 BookWise 메인 카드 리스트를 통해 콘텐츠를 발견하고 결제 페이지로 이동하는 흐름

1. 구매자는 BookWise 메인 화면에서 카드 형태로 정렬된 콘텐츠의 썸네일과 가격을 빠르게 훑는다.
2. 관심 카드를 클릭하면 상세 페이지로 이동해 설명·리뷰·미리보기를 확인하고 결제 유도 영역을 탐색한다.
3. "결제하기" 버튼을 누르면 외부 결제 제공 업체 페이지로 이동하여 결제를 진행한다.
4. 결제 완료 후에는 이메일 안내를 통해 다운로드 링크를 받고 관련 콘텐츠 추천을 확인한다.

## UX 디자인 원칙

### 핵심 원칙

- **직관성**: 카드 기반 콘텐츠 목록과 상세 페이지를 중심으로 핵심 행동(상품 확인, 결제 전환)에만 집중한 최소 메뉴 구조를 유지한다.
- **즉시성**: 필터·상태 변경·결제 이동 시 즉시 상태 안내를 제공하고 실패 시 외부 결제 페이지로 재진입 루트를 분명히 한다.
- **신뢰성**: 카드와 상세 페이지에서 가격·언어·세금을 명확히 표시하여 구매자가 외부 결제 흐름을 신뢰할 수 있도록 한다.
- **접근성**: 최소 항목 클릭으로 정보 접근이 가능하도록 하고, 키보드 내비게이션과 색 대비를 고려한 컬러 팔레트를 적용한다.

### 플랫폼 및 화면

- **플랫폼**: 반응형 웹 (Desktop / Tablet / Mobile)
- **핵심 화면**:
- 메인 상품 리스트 페이지
- 콘텐츠 카드·필터 영역
- 상품 상세 소개 페이지
- 결제 유도 영역

### 디자인 제약사항

- 기존 CMS나 마케팅 도구처럼 맞춤형 스토어 페이지 빌더는 초기 버전에서 제공하지 않는다.
- 결제·정산 화면은 민감 정보를 다루므로 WebAuthn 적용을 검토하되 복잡도를 고려해 최소 UI로 구성한다.

## UI 디자인 목표

### 주요 UI 요소

1. **콘텐츠 카드 그리드**
   - 외부 관리자 페이지에서 등록된 콘텐츠를 카드 형태로 노출하고, 제목·카테고리·가격·노출 상태를 빠르게 보여준다.
   - 각 카드는 눌렀을 때 상세 페이지로 이동하도록 하며, 카드 자체에 결제 버튼을 두어 외부 결제 페이지로 이동한다.
   - 카드 확장은 최소로 유지하며, 상태 배지(신규/할인/일시정지)만 표시해 복잡도를 줄인다.

2. **상품 상세 페이지**
   - 표지 이미지, 가격, 파일 유형, 콘텐츠 소개를 구분해서 보여주고, 상세 페이지 최상단에 단일 결제 버튼을配置한다.
   - 결제 버튼 아래에 가격 등 간단한 정보만 노출하고, 추천 상품·FAQ 섹션은 하단에 배치한다.
   - 구매에 필요한 링크(FAQ, 환불 정책)는 최소한으로 배치하여 결제 로직 집중한다.

3. **구매 흐름**
   - 상세 페이지에서 결제 버튼을 클릭하면 외부 결제 페이지로 이동하며, 이동 전 본문에 간단한 안내 문구만 표시한다.
   - 결제 이전에는 가격·통화·환율 정보만 표시하고, 추가 입력폼이나 선택지는 제공하지 않는다.
   - 결제 후 돌아오는 상태(주문 확인, 다운로드 안내)는 내부 로그로 간단히 트래킹하며 별도 UI는 제공하지 않는다.

### 인터랙션 패턴

- 마우스 오버 또는 탭 선택 시 콘텐츠 카드가 확대되고 미리보기 버튼이 노출된다.
- 결제 실패 시 어렵지 않게 다시 시도할 수 있도록 결제 단계 전체가 모달로 표시된다.
- 운영자는 콘텐츠 등록 중 파일 오류가 발생하면 파일 업로드 섹션에서 즉시 오류 메시지를 확인하고 재업로드한다.
- 리뷰 작성 또는 문의 발생 시 빠른 응답을 위해 푸시 메시지(웹 푸시 또는 이메일)를 보내고, 사용자가 알림을 끌 수 있도록 설정한다.

## 범위 외 (Out of Scope)

### 현재 버전에서 제외되는 기능

- **물리적 출판 연동**: 인쇄 및 배송 파트너 연동은 초기 버전에서 다루지 않는다.
- **구독형 멤버십 관리**: 월간/연간 구독 상품 모델은 별도 요구가 있으면 추후 고려한다.
- **AI 콘텐츠 생성 기능**: 생성형 AI를 통한 콘텐츠 제작 기능은 논의되지 않았으며 포함하지 않는다.
- **외부 마켓플레이스 동기화**: 다른 플랫폼과의 실시간 재고/가격 동기화 도구는 향후 과제로 둔다.
- **다국어 지원/로케일 시스템**: MVP에서는 단일 언어 UI만 제공하며 다국어 인터페이스와 로케일 스위칭은 추후 확장으로 둔다.
- **인증 및 사용자 세션**: MVP에서는 로그인/인증 기능을 제외하고 공개 페이지 중심으로 구성한다.
- **직접적인 DB 통합**: Lemon Squeezy API를 주기적으로 조회하는 구조이며 자체 DB 연동이나 마이그레이션은 포함하지 않는다.
- **리뷰/평점 시스템**: 리뷰 저장 및 관리 기능은 MVP 범위에서 제외한다. **추천 상품·FAQ는 포함**한다.

### 지원하지 않는 플랫폼

- 전용 네이티브 모바일 앱 (iOS/Android)
- 오프라인 키오스크 및 물리 부스 채널
- 외부 파트너 플랫폼에서 직접 접근하는 백오피스 API

### 기술적 제약

- BookWise MVP에서는 인증/사용자 세션 기능을 제외하고 단순히 공개된 콘텐츠 목록과 외부 결제 링크만 제공한다.
- 백엔드 데이터베이스를 직접 관리하지 않고, 외부 관리자 페이지의 데이터를 주기적으로 가져오는 방식으로 구현한다.
- 가격/정산 API는 컨테이너화된 서버리스 환경에서 운영되며 상태 정보를 외부 DB와 공유하지 않는다.
- 실시간 채팅·라이브 스트리밍 솔루션은 통합하지 않으며, 메시징은 이메일/웹훅 위주로 구성한다.
